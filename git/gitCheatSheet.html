Git Cheat Sheet*

CREATE LOCAL CHANGES BRANCHES & TAGS MERGE & REBASE COMMIT HISTORY UPDATE & PUBLISH UNDO
Clone an existing repository $ git clone ssh://user@domain.com/repo.git Changed files in your working directory $ git status List all existing branches $ git branch -av Merge <branch> into your current HEAD $ git merge <branch> Show all commits, starting with newest $ git log List all currently configured remotes $ git remote -v Discard all local changes in your working directory $ git reset --hard HEAD
Create a new local repository $ git init Changes to tracked files $ git diff Switch HEAD branch $ git checkout <branch> Rebase your current HEAD onto <branch> Don't rebase published commits $ git rebase <branch> Show changes over time for a specific file $ git log -p <file> Show information about a remote $ git remote show <remote> Discard local changes in a specific file $ git checkout HEAD <file>
Add all current changes to the next commit $ git add . Create a new branch based on your current HEAD $ git branch <new-branch> Abort a rebase $ git rebase --abort Who changed what and when in <file> $ git blame <file> Add new remote repository named <remote> $ git remote add <shortname> <url> Revert a commit (by producing a new commit with contrary changes) $ git revert <commit>
Add some changes in <file> to the next commit $ git add -p <file> Create a new tracking branch based on a remote branch $ git checkout --track <remote/branch> Continue a rebase after resolving conflicts $ git rebase --continue View change history of a file $ git log <filename> Download all changes from <remote>, but don't integrate into HEAD $ git fetch <remote> Reset your HEAD pointer to a previous commit ...and discard all changes since then $ git reset --hard <commit>
Commit all local changes in tracked files $ git commit -a Delete a local branch $ git branch -d <branch> Use your configured merge tool to solve conflicts $ git mergetool Download changes and directly merge/integrate into HEAD $ git pull <remote> <branch> ...and preserve all changes as unstaged changes $ git reset <commit>
Commit previously staged changes $ git commit Mark the current commit with a tag $ git tag <tag-name> Use your editor to manually solve conflicts and (after resolving) mark file as resolved $ git add <resolved-file> $ git rm <resolved-file> Publish local changes on a remote $ git push <remote> <branch> ...and preserve uncommitted local changes $ git reset --keep <commit>
Change the last commit Don't amend published commits $ git commit --amend Delete a branch on the remote $ git branch -dr <remote/branch>
Publish your tags $ git push --tags

This was done as an exercise in making a Bootstrap table and was created with the.table-condensed,.table-striped,.table-borderedand.table-hide classes.

The first three classes mentioned are used most often. I wanted to further enhance the look of the table and hide the empty cells. For this, I used the.table-hideclass. In order to hide the cells properly further styling must be done in CSS. First, theborder-collapsemust be changed from the default ofcollapsetoseparateandempty-cellsset tohide. (Read more about the border-collapse command on MDN.) The.table-borderedclass also had to be changed in CSS. The default of 1px is changed to 0px so the outer border is hidden.

CSS looks like this;


.table-hide {
  border-collapse: separate;
  empty-cells: hide;
}

.table-bordered {
  border: 0px solid #ddd;
}

My typical day-to-day GitHub commands

On a typical day I use a small subset of these commands. After cloning a repo, or creating a new git repo, the first thing to do is create a branch to work in,git checkout -b branch_name master. A brief run down to push code to the master branch after completing a new feature in the branch;

git status
lists the changed file(s)
git add
adds the current changes to the next commit
git commit -m " "
commits the local change(s) and adds a message about what was done between the quotes
git push
pushes the changes from branch_name to the local master branch
git checkout master
switches to the local master branch
git merge branch_name
merges the changed code into the local master branch
git push
pushes the local master branch to the remote master repo
